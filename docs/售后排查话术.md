# 售后排查话术

> 给卖家使用的标准化回复模板。
> 原则：简短、专业、不刺激买家、不承诺不该承诺的、引导运行 doctor.ps1。

---

## 场景 1：买家不会打开 PowerShell

**买家说**："我怎么打开那个蓝色的窗口？"

**话术**：
> 您好！打开方式很简单：
> 1. 打开我发给您的文件夹
> 2. 按住键盘 Shift 键不放
> 3. 在文件夹空白处点鼠标右键
> 4. 选择"在此处打开 PowerShell 窗口"
> 5. 如果没看到这个选项，也可以按键盘 Win+R，输入 powershell 回车

> 打开后把这段命令粘贴进去回车就行：
> `powershell -ExecutionPolicy Bypass -File .\install.ps1`

## 场景 2：买家不会获取 DeepSeek API Key

**买家说**："DeepSeek API Key 是什么？在哪里弄？"

**话术**：
> DeepSeek API Key 是您使用 AI 的"钥匙"，需要在 DeepSeek 官网免费注册获取：
> 1. 浏览器打开 https://platform.deepseek.com
> 2. 用手机号注册（免费）
> 3. 登录后左侧找到"API Keys"
> 4. 点"创建新 Key"
> 5. 复制保存（只显示一次！）

> 注意：API Key 是您自己的，我只帮您配置到本地，不要发给我。

## 场景 4：claude 命令不存在

**买家说**："安装完了但输入 claude 提示不存在"

**话术**：
> 这是 PATH 环境变量没刷新导致的，很常见。请尝试：
> 1. **关闭当前 PowerShell 窗口**
> 2. **重新打开一个** PowerShell
> 3. 再输入 `claude --version` 试试
>
> 如果还不行，请运行诊断脚本：
> `powershell -ExecutionPolicy Bypass -File .\doctor.ps1`
> 
> 把生成的 report.txt 发给我看看，我帮您排查。

---

## 场景 5：VS Code code 命令不存在

**买家说**："提示 code 命令未找到"

**话术**：
> code 命令是 VS Code 的命令行入口，需要在 VS Code 中手动开启：
> 1. 打开 VS Code
> 2. 按 Ctrl+Shift+P
> 3. 搜索 "Shell Command"
> 4. 选择 "Shell Command: Install 'code' command in PATH"
> 5. 完成后关闭所有终端窗口，重新打开即可
>
> 这不影响 Claude Code 使用，只是 VS Code 扩展安装这一步被跳过了。

## 场景 6：WSL 连接不上

**买家说**："wsl 命令用不了"

**话术**：
> WSL 是可选功能，不影响 Claude Code 在 Windows 上的使用。如果您不需要在 Linux 环境中用 Claude Code，可以忽略这个提示。
>
> 如果需要启用 WSL：
> 1. 以管理员身份打开 PowerShell
> 2. 运行：`wsl --install`
> 3. 重启电脑
>
> [WARN] WSL 安装可能需要较长时间且需要良好网络，这不是必须的。

## 场景 7：DeepSeek 401

**买家说**："提示 401 错误"

**话术**：
> 401 表示 API Key 不正确。请检查：
> 1. Key 是否完整复制（可能漏了开头或结尾的字符）
> 2. Key 是否已被删除（在 platform.deepseek.com 确认 Key 状态）
> 3. 重新到 platform.deepseek.com 创建新 Key
> 4. 运行 `.\configure-deepseek.ps1` 重新配置
>
> 如果仍然 401，请运行 doctor.ps1 生成报告发给我看看。

---

## 场景 8：DeepSeek 402

**买家说**："提示 402 错误"

**话术**：
> 402 表示您的 DeepSeek 账户余额不足。
> 
> 解决方案：
> 1. 访问 https://platform.deepseek.com
> 2. 进入"充值"页面
> 3. 充值后等待 2-3 分钟生效
> 4. 不需要重新配置，余额到账后直接可用
>
> [WARN] 这是 DeepSeek 平台的计费问题，与我们服务无关。

---

## 场景 9：DeepSeek 429

**买家说**："提示 429 错误"

**话术**：
> 429 表示请求太频繁被限流了。解决方法很简单：
> 1. **等待 3-5 分钟** 再试
> 2. 检查是否同时开了多个 claude 会话
> 3. 如果用了共享 Key，可能是别人也在用
>
> 一般等几分钟就恢复了。

## 售前问题模板

### "能试用吗？"
> 这是配置服务，不是软件产品。配置完成后您可以立即验证各项功能是否正常。有诊断报告可以确认每一步的状态。

### "支持 Mac 吗？"
> 当前支持 Windows 10 1809+/Windows 11 和 WSL (Ubuntu 20.04+)。系统要求：4GB+ 内存、64 位系统。Mac 用户请私聊确认。

### "会不会被封号？"
> 您使用的是自己的 DeepSeek 官方 API Key，通过官方接口调用。关于账户安全和使用规则，请以 DeepSeek 平台当前规则为准。我们的工具只做本地配置，不涉及任何违规操作。

### "以后更新要另外收费吗？"
> 不收费。Claude Code 更新您自己运行 `npm install -g @anthropic-ai/claude-code@latest` 就行。
