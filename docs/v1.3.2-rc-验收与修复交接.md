- Windows 本地路径验收，不使用 `\\wsl.localhost` / `\\wsl$`。
- 自动化验收不使用真实 DeepSeek API Key。

## 3. 已修复问题记录

### 3.1 压缩包临时路径未 BLOCK

- 等级: P1
- 失败现象:
  - `TEMP\Rar$EXa123\ClaudeDeepSeek` 返回 `WARN / IsBlocked=False`
  - `TEMP\7zO123456\ClaudeDeepSeek` 返回 `WARN / IsBlocked=False`
- 根因:
  - `Test-IsZipInternalPath` 使用 `Contains()` 做字面匹配。
  - marker 写成 `"\\rar$"`、`"\\7z"` 等双反斜杠字符串。
  - PowerShell 字符串中的反斜杠不是转义符，真实 Windows 路径只有单反斜杠，导致不命中。
- 修复提交:
  - `bb5d159` `fix(path): block archive temp extraction paths`
- 修复内容:
  - 路径规范化。
  - 路径段级正则匹配。
  - `Rar$...` 明确 BLOCK。
  - TEMP 下 `7z...` 明确 BLOCK。
  - TEMP 下 `.zip` / `Temp1_*.zip` 明确 BLOCK。
  - 普通 TEMP 仍为 WARN。
  - 非 TEMP 的 `D:\tools\7zip-helper\...` 不 BLOCK。

### 3.2 空 env 在 StrictMode 下恢复失败

- 等级: P1
- 失败现象:
  - `uninstall-config.ps1 -RestoreLatest -Yes -NonInteractive` 在沙盒恢复路径失败。
  - 错误: `The property 'Name' cannot be found on this object.`
- 触发链路:
  - `RemoveDeepSeekEnv` 后 `settings.json` 保留 `env = {}`。
  - `RestoreLatest` 前 `Show-StateAndCurrentConfig` 调用 `Get-DeepSeekConfigStatus`。
  - StrictMode 下访问空 PSCustomObject 的 `.PSObject.Properties.Name` 抛错。
- 修复提交:
  - `4433b64` `fix(config): handle empty env under strict mode`
- 修复内容:
  - 新增安全属性枚举 helper。
  - `Get-DeepSeekConfigStatus` 对空 env 返回未配置，不抛错。
  - `Get-ApiKeyFromConfig` 对空 env 返回 `$null`，不抛错。
  - 未关闭 StrictMode。
  - 未改变 remove/restore 核心语义。

### 3.3 安装流程 claude doctor 等待过久

- 等级: P2/P3 UX
- 用户反馈:
  - Native Install 成功后显示 `运行 claude doctor...`，很久之后才出现 `按回车键继续...`。
- 根因:
  - 安装流程同步等待 `claude doctor`，输出被捕获，用户看不到实时输出。
- 修复提交:
  - `369a9cf` `fix(ux): bound claude doctor quick check`
- 修复内容:
  - 安装流程中的 `Invoke-ClaudeDoctorSafe` 改为最多等待 30 秒。
  - 每 10 秒提示一次。
  - 超时或异常不阻断安装流程。

### 3.4 API smoke test 异常对象缺少 Response 属性

- 等级: P1
- 用户反馈:
  - Step 5/7 DeepSeek API 测试报错:
    - `在此对象上找不到属性“Response”。请确认该属性存在。`
- 根因:
  - `Test-DeepSeekApiAnthropic` 的 catch 直接访问 `$_.Exception.Response`。
  - 某些网络异常对象没有 `Response` 属性，StrictMode 下直接抛错。
- 修复提交:
  - `e708863` `fix(api): handle response-less smoke test errors`
- 修复内容:
  - 增加安全属性枚举。
  - 异常对象先判断是否存在 `Response` 属性。
  - 无 `Response` 的网络错误返回结构化错误，不再未预期崩溃。
  - API 响应解析也避免直接访问不存在的 `content` / `choices` 属性。
  - `scripts/check.ps1` 增加本地关闭端口回归，不请求 DeepSeek。

### 3.5 doctor.ps1 中 claude doctor 仍等待过久

- 等级: P2/P3 UX
- 用户反馈:
  - 环境诊断工具里的 `正在运行 claude doctor...` 仍然等很久。
- 根因:
  - `doctor.ps1` 中另有一处直接调用 `Invoke-CommandSafe -Command "claude" -Arguments @("doctor")`。
  - 该处没有使用安装流程中已修复的 30 秒上限逻辑。
- 修复提交:
  - `6e01a04` `fix(doctor): bound claude doctor diagnostics`
- 修复内容:
  - `doctor.ps1` 中 `claude doctor` 改为快速诊断，最多等待 30 秒。
  - 每 10 秒输出进度提示。
  - 超时或非零退出码记为 WARN，不阻断后续诊断。
  - `scripts/check.ps1` 增加静态回归检查。

### 3.6 partial state 缺 installedAt 导致恢复/卸载崩溃

- 等级: P1
- 用户反馈:
  - 双击“恢复或卸载配置”后报错：`在此对象上找不到属性“installedAt”。`
- 根因:
  - 旧版或残缺 `state.json` 缺少 `installedAt`。
  - `uninstall-config.ps1` 在 StrictMode 下直接访问 `$state.installedAt`。
- 修复提交:
  - `ab6a688` `fix(state): tolerate partial installer state`
- 修复内容:
  - 新增 `Get-CcdiStateValue`。
  - 缺字段显示 `(未记录)` 或 `(未知)`。
  - 新增 `-ShowStatusOnly` 作为无交互状态展示回归入口。

### 3.7 release 用户模拟卡在 claude doctor

- 等级: P1 验收阻断
- 失败现象:
  - `simulate-user-release.ps1` 多次运行诊断入口，每次触发本机真实 `claude doctor`，在部分机器上每次等待 30 秒。
- 修复提交:
  - `e83c866` `fix(validation): harden release user simulation`
- 修复内容:
  - `doctor.ps1` 在 `CCDI_TEST_MODE=1` 下跳过真实 `claude doctor`。
  - release 用户模拟补 partial state 回归。
  - `scripts/check.ps1` 增加静态回归。

### 3.8 DeepSeek API 200 但响应解析失败

- 等级: P1
- 失败现象:
  - 真实 API smoke 返回 HTTP 200，但函数判定失败。
- 根因:
  - `deepseek-v4-flash` 可能先返回 thinking block，text block 在后。
  - 旧逻辑只读取 `content[0].text`。
- 修复提交:
  - `e83c866` `fix(validation): harden release user simulation`
- 修复内容:
  - smoke test `max_tokens` 提高到 64。
  - 遍历 `content[]`，查找第一个带 `text` 的 block。

### 3.9 中文路径误报风险

- 等级: P2 UX
- 用户反馈:
  - release ZIP 文件名本身含中文，默认解压后路径含中文，却提示“路径包含中文字符，可能导致 CMD/PowerShell 解析异常”。
- 修复提交:
  - `29b6f6d` `fix(path): allow Chinese project paths`
- 修复内容:
  - 中文路径本身不再触发 WARN。
  - 空格、特殊字符、TEMP、ZIP 预览临时目录仍保留风险提示。
  - 文档中“路径不要有中文”改为“路径不要有空格、特殊字符”。
