# 发布前验收清单

> 开发者用。每次 release 打包前执行。常规发布优先使用统一入口，不再手工重复拼接零散命令。

## 1. 前置确认

```powershell
git fetch origin --prune
git checkout release/v1.3.2-rc
git pull --ff-only origin release/v1.3.2-rc
git status --short
git log --oneline --decorate -8
```

要求：

- 当前分支是 `release/v1.3.2-rc`。
- 工作区没有意外源码改动。
- 不提交 `.sandbox/`、`logs/`、`backup/`、`reports/`、`release/`、`report.txt`。

## 2. 快速冒烟

小改动后先跑：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Smoke
```

覆盖：

- `git diff --check`
- `git diff --cached --check`
- `scripts/check.ps1` PowerShell 5.1
- `scripts/check.ps1` pwsh（如果本机可用）
- 路径风险矩阵
- StrictMode 回归
- API 异常回归
- `.cmd` 编码
- 安装安全结构检查

## 3. 全量源码验收

修改配置、诊断、安装、恢复/卸载逻辑后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Full
```

覆盖：

- 快速冒烟全部内容
- PowerShell 全量 AST parse
- `scripts/ux-check.ps1`
- `scripts/install-decision-matrix.ps1`（Windows 安装决策树 mock 验收，10 个场景）
- Start-Here TestSafe 沙盒链路
- repair-deps TestSafe
- Start-Here -FixDeps TestSafe
- doctor ShareSafe + SkipApiTest
- 真实 settings.json 前后 hash/length 不变

## 4. 交付前验收

准备给用户发布前运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Release -Version "1.3.2"
```

覆盖：

- release ZIP 构建
- ZIP 不包含开发者脚本（build-release.ps1、simulate-user-release.ps1、package-release.ps1）
- ZIP 白名单内容检查
- API Key 源码和 staging 扫描
- `.ps1` BOM、`.cmd` ASCII、`.sh` LF
- 解压到含中文和空格的目录
- 中英文入口模拟
- ShellExecute 双击启动器模拟（环境不支持时 SKIP）
- Windows 场景矩阵汇总（windows-scenario-matrix.ps1）
- 诊断、恢复/卸载、删除配置、恢复配置
- API mock: 200/401/402/429/503/timeout/dns
- 缺文件启动器提示
- partial state 缺 `installedAt` 回归

## 5. 最终完整验收

正式交付前推荐直接运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode All -Version "1.3.2" -RequireClean
```

通过标准：

- 所有步骤 exit code 为 0。
- `validate.ps1` 最后显示 `Validation PASSED`。
- 真实 `%USERPROFILE%\.claude\settings.json` 未变化。
- `git status --short` 没有源码未提交改动（All 模式自动强制）。
- 记录 release ZIP SHA256 到 `docs/release-artifacts.md`。

手工兜底命令（发布前最终确认）：

```powershell
git diff --check
git diff --cached --check
git status --short
```

## 6. 最终 SHA256

打包通过后记录到 `docs/release-artifacts.md`：

```powershell
Get-Content .\release\ClaudeCode-DeepSeek-本地配置助手-v1.3.2.zip.sha256
```

将 SHA256 填入 `docs/release-artifacts.md` 的对应版本条目。

## 7. 安装决策树 Mock 验收

独立运行安装决策树 mock 矩阵：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\install-decision-matrix.ps1
```

覆盖 10 个安装决策分支（不真实安装任何软件）。

## 8. Windows 场景矩阵

生成场景覆盖矩阵报告：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\windows-scenario-matrix.ps1 -Version "1.3.2"
```

输出 `reports/windows-scenario-matrix-report-YYYYMMDD-HHMMSS.txt`。

## 9. 手动抽查

自动化通过后，必要时在解压目录人工双击：

- `开始安装.cmd`
- `Start-Install.cmd`
- `一键诊断.cmd`
- `Run-Diagnostics.cmd`
- `恢复或卸载配置.cmd`
- `Restore-Config.cmd`

观察：

- 不闪退。
- 不出现 ParserError。
- 中文显示正常。
- 失败提示能告诉用户下一步。

## 10. 故意泄露测试

只有改动打包扫描逻辑时才需要手动运行：

```powershell
$fake = "sk-" + "leaktest0123456789abcdef0123456789"
Set-Content .\leak-test.md "DEEPSEEK_API_KEY=$fake"
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-release.ps1 -SkipSha256
Remove-Item .\leak-test.md
```

预期：

- 打包失败。
- 提示疑似真实 API Key。
- `leak-test.md` 必须删除，不能提交。

## 11. 环境限制记录

可以记录为环境限制但不能静默忽略：

- 本机无 WSL 或 WSL 状态异常。
- 源工作区 `bash -n install_wsl.sh` 因 Windows/WSL CRLF 行为失败。
- VS Code 扩展未安装。

不能作为环境限制放过：

- 真实 settings 被修改。
- release ZIP 含敏感信息。
- `.cmd` 或 `.ps1` 编码失败。
- 用户入口崩溃。
- 恢复/卸载配置崩溃。
