# 验收与交接体系

> 面向后续维护者。目标是复用已有验收资产，统一运行入口，沉淀历史问题回归，避免每次修复后重新发明验证流程。

## 2. 统一验收入口

统一入口是：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Smoke
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Full
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Release -Version "1.3.2"
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode All -Version "1.3.2" -RequireClean
```

`validate.ps1` 只做编排，不替代已有脚本。失败时会输出失败步骤、耗时和底层错误。每次运行都会记录真实 `%USERPROFILE%\.claude\settings.json` 的存在性、长度和 SHA256，并在结束时确认未变化。

### 模式说明

| Mode | 用途 | 覆盖范围 | Clean 策略 | 典型耗时 |
|---|---|---|---|---|
| `Smoke` | 小改动后快速判断有没有破坏核心 | `git diff --check`、`git diff --cached --check`、`scripts/check.ps1` PowerShell 5.1、可用时 pwsh | 不强制干净（允许开发中的临时文件） | 短 |
| `Full` | 源码侧全量验收 | `Smoke`、全量 AST parse、`ux-check.ps1`、Start-Here/repair-deps/doctor 沙盒链路 | 不强制干净 | 中 |
| `Release` | 交付前验收 | 打包 ZIP、release 用户全链路模拟 | 不强制干净（release/ 产物在 .gitignore 内） | 中到长 |
| `All` | 完整交付判断 | `Full` + `Release` | **强制干净**（git status --short 必须为空） | 长 |

### Clean 策略说明

- Smoke / Full / Release 默认不强制工作区干净，避免开发中被 release/ 或临时文件打断。
- All 模式必须强制最终工作区干净（等效于 `-RequireClean`）。
- 用户显式传入 `-RequireClean` 时，无论什么 Mode 都强制干净。
- 工作区不干净时输出具体文件列表并 exit 1。
- Smoke 级别已包含 `git diff --check` 和 `git diff --cached --check`（检查冲突标记和空白错误）。
- 手工兜底命令（发布前最终确认）：
  ```powershell
  git diff --check
  git diff --cached --check
  git status --short
  ```

## 3. 快速冒烟验收

适用场景：

- 修改单个脚本后。
- 修复小 bug 后。
- 提交前快速确认。

运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Smoke
```

必须通过：

- PowerShell 语法解析。
- bootstrap 导出。
- 路径风险矩阵。
- empty env StrictMode。
- partial state missing `installedAt`。
- API response-less exception。
- `claude doctor` TestSafe 跳过。
- `.cmd` 编码。
- 安装安全结构检查。

## 4. 全量源码验收

适用场景：

- 修改配置写入、状态读取、诊断、安装逻辑。
- 修复用户阻断 bug。
- 合并多个修复后。

运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Full
```

覆盖：

- 快速冒烟全部内容。
- PowerShell 全量 AST parse。
- `ux-check.ps1` 的文档引用、真实配置隔离、配置合并、状态文件、脱敏。
- 沙盒 Start-Here TestSafe。
- 沙盒 repair-deps TestSafe。
- 沙盒 Start-Here -FixDeps TestSafe。
- 沙盒 doctor ShareSafe + SkipApiTest。

判定：

- 任一失败均不得发布。
- 如果失败来自本机环境限制，应记录清楚；除非明确是环境限制，否则先按项目问题排查。

## 5. 全链路用户体验模拟

适用场景：

- 交付前。
- 修改入口、打包、诊断、恢复/卸载、文档、报告输出后。

运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\validate.ps1 -Mode Release -Version "1.3.2"
```

底层会运行：

- `scripts/package-release.ps1`
- `scripts/simulate-user-release.ps1`

覆盖：

- 构建 release ZIP。
- 解压到含中文和空格的临时目录。
- PowerShell 5.1 AST parse。
- `.ps1` BOM 和 `.cmd` ASCII。
- 中英文安装入口取消流程。
- Start-Here TestSafe 完整流程。
- 中英文诊断入口。
- 中英文恢复/卸载入口。
- `uninstall-config.ps1 -ListBackups`。
- `RemoveDeepSeekEnv` 保留自定义配置。
- `RestoreLatest`。
- `DeleteSettings` 后再恢复。
- 无 API Key 的非交互配置失败提示。
- 损坏 JSON 重建。
- API mock: 200/401/402/429/503/timeout/dns。
- 缺文件启动器提示用户完整解压 ZIP。
- release ZIP 内 partial state 缺 `installedAt` 回归。
- 运行输出和日志不泄露完整假 Key。

## 6. 历史问题回归清单

| 问题 | 表现 | 当前解决方式 | 验收方式 | 自动化 |
|---|---|---|---|---|
| ZIP 预览/压缩包临时路径未 BLOCK | 从 `Rar$...`、`7z...`、`Temp1_*.zip` 运行导致丢文件 | `Test-IsZipInternalPath` 路径段检测 | `scripts/check.ps1` 路径矩阵 | 是 |
| 中文路径误报 WARN | 用户默认解压中文 ZIP 名后看到风险警告 | 中文路径本身不再计入风险 | `Chinese only path: INFO` | 是 |
| empty env StrictMode 崩溃 | `env = {}` 时恢复/诊断抛属性不存在 | 安全枚举 JSON 属性 | `scripts/check.ps1` empty env | 是 |
| partial state 缺 `installedAt` 崩溃 | 恢复/卸载配置入口报 `installedAt` 属性不存在 | `Get-CcdiStateValue` 安全读取 | `scripts/check.ps1` + release 模拟 | 是 |
| API 异常缺 `Response` 崩溃 | 网络异常时 catch 中访问不存在属性 | 异常属性安全判断 | `scripts/check.ps1` 本地关闭端口 | 是 |
| API 200 但无 text 误判失败 | DeepSeek 返回 thinking block 在前，text 在后 | 遍历 content blocks 查找 `text` | 真实 API smoke 可手动运行；mock 由 simulate 覆盖 | 部分 |
| 安装流程 `claude doctor` 等太久 | 用户卡在快速诊断 | 30 秒上限，失败不阻断 | `scripts/check.ps1` 静态检查 | 是 |
| doctor 中 `claude doctor` 等太久 | 诊断入口多次等待 | TestSafe 跳过，真实诊断 30 秒上限 | `scripts/check.ps1` + release 模拟 | 是 |
| `-ListBackups` 输出被吞 | 用户看不到备份列表 | 不 suppress 输出，按文件名排序 | `scripts/check.ps1` + release 模拟 | 是 |
| Copy-Item 保留 LastWriteTime 导致恢复顺序错 | 最新备份判断错误 | 按备份文件名排序 | `scripts/check.ps1` | 是 |
| `.cmd` 中文乱码 | CMD 解析 UTF-8 中文失败 | `.cmd` 纯 ASCII，无 BOM | `scripts/check.ps1` + release 模拟 | 是 |
| `.ps1` PowerShell 5.1 乱码 | 无 BOM 导致中文脚本解析失败 | release staging 自动补 BOM | `build-release.ps1` + release 模拟 | 是 |
| 真实 settings 被污染 | 自动化写入真实用户配置 | `CCDI_TEST_USERPROFILE` 沙盒 + hash 对比 | `validate.ps1` + `ux-check.ps1` | 是 |
| release ZIP 包含内部文件或敏感信息 | 上传错误产物 | 白名单打包 + source/staging 扫描 | `build-release.ps1` | 是 |

## 7. 交付前验收标准

交付前必须满足：

- `git status --short` 无源码未提交改动。
- `scripts/validate.ps1 -Mode All -Version "1.3.2" -RequireClean` 通过。
- release ZIP 重新生成并记录 SHA256 到 `docs/release-artifacts.md`。
- release ZIP 不包含 `.git`、`logs`、`backup`、`reports`、`release`、`.gitignore`、`CLAUDE.md`、`build-release.ps1`、`simulate-user-release.ps1`、`package-release.ps1`、`report.txt`。
- 自动化验收未修改真实 `%USERPROFILE%\.claude\settings.json`。
- 敏感扫描未发现真实 API Key。
- 售后诊断链路 `doctor.ps1 -ShareSafe` 可用。

可以记录为环境限制但不得静默忽略：

- 源工作区 `bash -n install_wsl.sh` 因 Windows/WSL CRLF 行为失败。release ZIP 内 `install_wsl.sh` 仍必须 LF。
- 本机无 WSL 或 WSL 发行版状态异常。Windows 主流程不能因此失败。
- 本机未安装 VS Code 扩展。诊断可 WARN，不阻断发布。

## 8. 售后排查流程

统一入口：

```powershell
powershell -ExecutionPolicy Bypass -File .\doctor.ps1 -ShareSafe -NoOpenReport
```

让用户提供：

- 根目录 `report.txt`。
- 问题发生步骤。
- 截图时遮住 API Key。

不要让用户提供：

- 完整 API Key。
- `backup/`。
- `logs/`。
- `reports/full-report-*`。
- `.claude/settings.json` 原文。

判断优先级：

- `API Key 401`：Key 错或已删除。
- `402`：余额/计费。
- `429`：限流。
- `5xx`：DeepSeek 官方波动。
- `settings.json JSON 格式无效`：让用户运行 `configure-deepseek.ps1` 重建。
- `claude 命令未找到`：先重开终端，再看 PATH/安装状态。
- `ZIP 临时目录 BLOCK`：让用户完整解压。

## 9. 后续维护规则

修改代码后：

- 小改动：跑 `validate.ps1 -Mode Smoke`。
- 配置读写、状态文件、诊断、安装逻辑：跑 `validate.ps1 -Mode Full`。
- 入口、打包、文档、恢复/卸载、报告输出：跑 `validate.ps1 -Mode Release`。
- 发布前：跑 `validate.ps1 -Mode All`。

修复 bug 后：

- 优先把复现步骤加入 `scripts/check.ps1` 或 `scripts/simulate-user-release.ps1`。
- 如果不能自动化，加入 `docs/用户体验验证清单.md` 或 `docs/测试清单.md`。
- 更新本文件的“历史问题回归清单”。
- 不要只写总结，不加回归。

新增功能后：

- 新入口：补 `.cmd` 映射、编码检查、release 模拟。
- 新配置字段：补 merge/remove/restore 回归。
- 新报告字段：补脱敏检查。
- 新 release 文件：同步 `build-release.ps1` 白名单和 `ux-check.ps1` 白名单检查。
